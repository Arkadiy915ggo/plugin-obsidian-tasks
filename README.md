# Rule Based Daily Tasks

MVP Obsidian-плагина для генерации задач в daily note на основе правил под задачами в отдельных заметках.

## Формат задач

По умолчанию плагин читает markdown-файлы из папки `Task Sources`.

```md
- [ ] Тренировка
  rule: 3 раза в неделю

- [ ] Проверить финансы
  правило: 2 раза в месяц

- [ ] Позвонить врачу
  rule: 2026-07-15

- [ ] Подготовиться к отпуску
  rule: 2026-07-10..2026-07-20

- [ ] Почитать книгу
  schedule: от 2 до 3 раз в неделю
```

Правило должно быть на следующей вложенной строке под задачей. Плагин понимает эти имена поля:

- `rule:`
- `schedule:`
- `правило:`
- `расписание:`

Двоеточие может быть одинарным или двойным: `rule: 3x/week` и `rule:: 3x/week` оба подходят.

Задача должна быть обычным markdown-checkbox:

```md
- [ ] Название задачи
  rule: 3 раза в неделю
```

Вложенный вариант с маркером у правила тоже поддерживается:

```md
- [ ] Название задачи
  - rule: 3 раза в неделю
```

## Поддерживаемые правила

### Несколько раз в неделю

```md
- [ ] Тренировка
  rule: 3 раза в неделю
```

Поддерживаемые формы:

- `3 раза в неделю`, `3x/week`
- `3 раза/неделю` не требуется, но короткий английский вариант `3x/week` работает
- `3 times week`

Неделя считается ISO-неделей: с понедельника по воскресенье. Если указано `3 раза в неделю`, плагин выберет 3 даты внутри этой недели.

### Диапазон количества в неделю

```md
- [ ] Почитать книгу
  rule: от 2 до 3 раз в неделю
```

Поддерживаемые формы:

- `от 2 до 3 раз в неделю`
- `2-3 раза в неделю`
- `2–3 раза в неделю`
- `2—3 раза в неделю`

Для каждой задачи и недели количество выбирается детерминированно внутри диапазона. То есть в одну неделю может быть 2 раза, в другую 3, но при повторной генерации той же недели результат не прыгает случайно.

### Несколько раз в месяц

```md
- [ ] Проверить финансы
  правило: 2 раза в месяц
```

Поддерживаемые формы:

- `2 раза в месяц`
- `2x/month`
- `2 times month`
- `от 2 до 4 раз в месяц`
- `2-4 раза в месяц`

Месяц считается календарным месяцем. Если указано `2 раза в месяц`, плагин выберет 2 даты внутри месяца и постарается разнести их по месяцу, а не поставить рядом.

### Точная дата

```md
- [ ] Позвонить врачу
  rule: 2026-07-15
```

Задача появится только в daily note за эту дату. Формат даты сейчас только `YYYY-MM-DD`.

### Диапазон дат

```md
- [ ] Подготовиться к отпуску
  rule: 2026-07-10..2026-07-20
```

Задача появится каждый день внутри диапазона, включая начало и конец.

Поддерживаемые разделители диапазона:

- `2026-07-10..2026-07-20`
- `2026-07-10 - 2026-07-20`
- `2026-07-10 до 2026-07-20`
- `2026-07-10 to 2026-07-20`
- `2026-07-10–2026-07-20`
- `2026-07-10—2026-07-20`

Если даты указаны в обратном порядке, плагин сам поменяет начало и конец местами.

## Как выбираются даты

Для недельных и месячных правил плагин детерминированно выбирает даты внутри периода и старается распределять их равномерно. Например, правило `3 раза в неделю` всегда даст ровно 3 дня в ISO-неделе для конкретной задачи, без случайного сбивания всех дней подряд.

Выбор зависит от:

- пути source-файла;
- номера строки задачи;
- текста задачи;
- текста правила;
- недели или месяца.

Из-за этого результат стабилен при повторной генерации, но может измениться, если переименовать задачу, перенести ее в другой файл или вставить новые задачи выше нее.

## Daily notes

Плагин считает заметку daily note, если:

- файл находится в папке из настройки `Daily notes folder`;
- имя файла соответствует формату даты из настройки `Daily note date format`, по умолчанию `YYYY-MM-DD`.

При создании daily note плагин добавляет или обновляет блок:

```md
## Generated tasks

- [ ] Тренировка
- [ ] Проверить финансы
```

Также есть команда `Generate tasks for current daily note`.

Если в старой daily note уже есть блок с комментариями `<!-- rule-based-daily-tasks:start -->` и `<!-- rule-based-daily-tasks:end -->`, следующая генерация заменит его на чистый блок без комментариев.

## Разработка

```bash
npm install
npm run build
```

Для установки в Obsidian папка проекта должна находиться в `.obsidian/plugins/rule-based-daily-tasks` внутри vault. После этого плагин можно включить в настройках Community plugins.
